Room Numbering v2 (**Session Counts & Room Indicators**, Jul 2026). Every tutor owns a **frozen block** of consecutive Zoom breakout-room numbers sized `assigned students + buffer`, where the buffer scales with tutor count (**3** for ≤4 tutors · **2** for 5–8 · **1** for 9+). The header **room-count badge** is the sum of all blocks. The algorithm lives in `roomNumbering.js` and is unit-tested (`roomNumbering.test.js`).

### Rules (normative)

1. **Initial assignment** — reserve each tutor a block of `assigned + buffer`; blocks then **freeze**. Room count = sum of blocks.
2. **Assign** — a student takes the lowest free number in their tutor's block; freed numbers **recycle**.
3. **Move** — free the old number, reassign in the destination block; no one else renumbers.
4. **Absence** — no change; the student keeps their number and the room stays empty.
5. **New student** — assign per rule 2 (recycled numbers included).
6. **Tutor joins mid-session** — inherits students in place; no new block, no renumbering.
7. **Block full → borrow** — take the lowest free number from any other block; the row shows the **out-of-range** indicator (warning tint). Rooms ≥ students holds by construction, so a borrow always succeeds.

### Badge states

- **Room count — waiting**: tertiary (muted) tint + `—` before the lead runs assignment (tooltip *“Assign students to tutors to size breakout rooms”*).
- **Room count — active**: secondary tint + frozen total (tooltip *“Rooms to create in Zoom (includes each tutor's buffer)”*).
- **Student room number**: secondary tint; **out-of-range** rows use warning tint (color-only) when the number is borrowed.
