# PLUS Design System Documentation

This directory contains all documentation for the PLUS design system, organized by type.

## Structure

```
docs/
├── guidelines/              # Core guidelines and reference documents
│   ├── README.md           # Guidelines overview
│   ├── coding-standards.md # Project rules and coding standards
│   ├── token-reference.md  # Complete token reference
│   └── terminology.md     # UI component types and terminology
│
├── components/             # Component documentation
│   ├── README.md          # Components overview
│   └── COMPONENTS.md      # Component library API documentation
│
├── tokens/                 # Token system documentation
│   └── README.md          # Token system architecture
│
├── DESIGN_PATTERNS.md     # Design patterns and examples
├── DEV_STANDARDS.md       # Development standards
├── DESIGN_TOKENS.md       # Legacy design tokens documentation
└── FIGMA_DESIGN_SYSTEM.md # Figma integration guide
```

## Quick Start

### For Designers
1. Start with `guidelines/README.md` for overview
2. Reference `guidelines/terminology.md` to identify component type
3. Reference `guidelines/token-reference.md` to select tokens
4. Reference `guidelines/coding-standards.md` for code patterns

### For Developers
1. Review `guidelines/coding-standards.md` for project rules
2. Reference `components/COMPONENTS.md` for component APIs
3. Check `tokens/README.md` for token system details
4. Review `DESIGN_PATTERNS.md` for design patterns

### For AI Agents (Cursor)
1. Always consult `guidelines/` files when generating code
2. Use `guidelines/terminology.md` when designing specific UI types
3. Use `guidelines/token-reference.md` when selecting tokens
4. Follow `guidelines/coding-standards.md` for code style
