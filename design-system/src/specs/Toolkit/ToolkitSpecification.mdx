# Toolkit

Organism-level specs for scheduling (pre-session), live classroom UI (in-session), and reflection flows (post-session). This page is the **single scrollable overview**; every Toolkit component also has its **own docs page** in the sidebar under **Specs → Toolkit →** phase → category → component.

## How documentation is organized

- **One CSF file per component**: `*.stories.jsx` next to implementation or local demo wrappers.
- **One MDX docs page per CSF file**: sibling `*.mdx` with **Overview**, **Variants** (only when the story file exports `Variants`), and **Interactive playground** — same pattern as **Profile** and **Components** docs (`sb-ds-component-docs--page`, inline canvases, `Resources` → GitHub).
- **Sidebar**: Stories use `tags: ['!dev', '!autodocs']` so canvases do not clutter the tree; navigation is MDX-first.
- **Merged Storybook titles**: Some modal/page families share one Storybook `title` across multiple files (e.g. several entries under one modal family). Each MDX uses `<Meta of={…} name="Toolkit Docs — …">` so docs pages stay distinct.
- **Regenerate sibling MDX** after adding or renaming stories: run `node scripts/toolkit-spec-docs.mjs` from the repo root.

Under each phase, specs follow **Elements**, **Cards**, **Tables**, **Modals**, **Sections**, and **Pages** (when present).
