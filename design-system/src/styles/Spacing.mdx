# Spacing

The spacing system is a two-tier token architecture: a **primitive scale** (raw
step values, never used directly) and **semantic per-context tokens** (what you
actually reference in code). Every gap, padding, radius, and stroke in the
design system resolves to one of these tokens — never a raw pixel value.

## Primitive scale

Primitives live in `design-system/src/tokens/_primitives.scss` and correspond to
the **size / primitive** variables in Figma (Spacing/Small, Spacing/Medium,
Spacing/Large). They are the base steps that semantic tokens are built from.
**Do not use them directly** — pick the semantic token for your context instead.

| Group | Token | Value |
|-------|-------|-------|
| Small | `--size-spacing-small-space-000` | 0px |
| Small | `--size-spacing-small-space-025` | 2px |
| Small | `--size-spacing-small-space-050` | 4px |
| Small | `--size-spacing-small-space-075` | 6px |
| Small | `--size-spacing-small-space-100` | 8px |
| Small | `--size-spacing-small-space-150` | 10px |
| Medium | `--size-spacing-medium-space-200` | 12px |
| Medium | `--size-spacing-medium-space-300` | 16px |
| Medium | `--size-spacing-medium-space-400` | 20px |
| Medium | `--size-spacing-medium-space-500` | 24px |
| Large | `--size-spacing-large-space-600` | 32px |
| Large | `--size-spacing-large-space-650` | 36px |
| Large | `--size-spacing-large-space-700` | 40px |
| Large | `--size-spacing-large-space-800` | 48px |
| Large | `--size-spacing-large-space-900` | 64px |
| Large | `--size-spacing-large-space-1000` | 80px |

Larger steps (`1200`–`5000`, xlarge) exist for page-scale layout, alongside
border-radius primitives (`--size-border-radius-radius-*`) and stroke-width
primitives (`--size-border-stroke-stroke-*`).

## Semantic tokens

Semantic tokens (`design-system/src/tokens/_spacing_semantics.scss`, matching
the **size / semantics** Figma collection) name spacing by *where it is used*,
not by its value. Each context provides padding, gap, radius, and border tokens
in `sm` / `md` / `lg` steps:

| Context | Token pattern | Used for |
|---------|--------------|----------|
| Element | `--size-element-pad-{x\|y}-{sm\|md\|lg}`, `--size-element-gap-{xs\|sm\|md\|lg}` | Buttons, inputs, chips, and their internal spacing |
| Card | `--size-card-pad-{x\|y}-{sm\|md\|lg}`, `--size-card-gap-{sm\|md\|lg}` | Card padding and spacing between items inside a card |
| Section | `--size-section-pad-{x\|y}-{sm\|md\|lg}`, `--size-section-gap-{sm\|md\|lg}` | Page sections and spacing between major blocks |
| Modal | `--size-modal-pad-{x\|y}-{sm\|md\|lg}`, `--size-modal-gap-{sm\|md\|lg}` | Modal body padding and content spacing |
| Surface | `--size-surface-pad-{x\|y}`, `--size-surface-gap-{sm\|md\|lg}` | Outermost page surface |
| Table | `--size-table-cell-{x\|y\|gap}` | Table cell padding and cell content gaps |

Radius and stroke follow the same pattern: `--size-card-radius-{sm|md|full}`,
`--size-element-radius-{sm|md|lg|full}`, `--size-element-stroke-{sm|md|lg|xl}`, etc.

## Which tier belongs where

Spacing is layered from the page inward. Choose the token tier that matches the
layer you are working in:

| Layer | Token tier | Example |
|-------|-----------|---------|
| Page / surface | `--size-surface-*` | Outer page padding |
| Container | `--size-surface-container-*` | Grouped content regions on the surface |
| Section | `--size-section-*` | `gap: var(--size-section-gap-md)` between major page blocks (e.g. two Cards) |
| Card / modal | `--size-card-*`, `--size-modal-*` | Card padding, modal body spacing |
| Element | `--size-element-*` | `gap: var(--size-element-gap-md)` between headers and text inside a Card |

Two rules from the layout pattern guide:

- `gap: var(--size-section-gap-md)` — spacing **between major page blocks** (like two different Cards).
- `gap: var(--size-element-gap-md)` — spacing **between elements inside a Card** (like headers and text).
- **Never** use raw pixel strings like `gap: '16px'`.
