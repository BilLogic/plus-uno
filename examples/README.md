# Reference Prototypes

This directory contains reference prototypes that demonstrate PLUS design system patterns and best practices.

## Purpose

Reference prototypes serve as:
- Training examples for the Cursor agent
- Visual reference for design patterns
- Code examples for common use cases
- Validation of design system consistency

## Prototype Structure

Each prototype should include:
- `index.html` - Main HTML file
- `styles.css` - Component-specific styles (if needed)
- `script.js` - JavaScript functionality (if needed)
- `README.md` - Documentation of the prototype

## Reference Prototypes

### 1. Button Showcase
**Purpose**: Demonstrates all button variants, styles, and sizes
**Location**: `examples/button-showcase/`
**Key Patterns**:
- Button variants (filled, outline, tonal, text)
- Button styles (primary, secondary, success, etc.)
- Button sizes (small, default, large)
- Button states (default, hover, active, disabled)

### 2. Form Elements
**Purpose**: Demonstrates form input patterns
**Location**: `examples/form-elements/`
**Key Patterns**:
- Text inputs
- Textareas
- Checkboxes
- Radio buttons
- Form validation
- Form layout

### 3. SMART Components
**Purpose**: Demonstrates SMART competency area components
**Location**: `examples/smart-components/`
**Key Patterns**:
- Status indicators
- Competency area pills
- SMART cards
- Status tags

### 4. Card Layouts
**Purpose**: Demonstrates card-based layouts
**Location**: `examples/card-layouts/`
**Key Patterns**:
- Basic cards
- SMART cards
- Card grids
- Card interactions

### 5. Dashboard Layout
**Purpose**: Demonstrates complex dashboard layout
**Location**: `examples/dashboard-layout/`
**Key Patterns**:
- Multi-column layouts
- Responsive grids
- Component composition
- Data visualization placeholders

## Creating New Prototypes

When creating a new prototype:

1. Create a new directory in `examples/`
2. Include all necessary files
3. Use design tokens consistently
4. Follow PLUS design patterns
5. Include documentation
6. Test across browsers and screen sizes

## Usage

Reference prototypes can be:
- Opened directly in a browser
- Used as code examples
- Referenced in Cursor conversations
- Used for design system validation

## Notes

- Prototypes should be self-contained
- Use CDN links for dependencies when possible
- Document any special requirements
- Keep prototypes simple and focused
- Update prototypes as design system evolves
