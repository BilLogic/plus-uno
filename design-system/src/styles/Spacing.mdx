The spacing system is a two-tier token architecture: a **primitive scale** (raw
step values, never used directly) and **semantic per-context tokens** (what you
actually reference in code). Every gap, padding, radius, and stroke in the
design system resolves to one of these tokens — never a raw pixel value.

### Primitive scale

Primitives live in `design-system/src/tokens/_primitives.scss` and correspond to
the **size / primitive** variables in Figma (Spacing/Small, Spacing/Medium,
Spacing/Large). They are the base steps that semantic tokens are built from.
**Do not use them directly** — pick the semantic token for your context instead.

<SpacingStories.PrimitiveScale />

Larger steps (`1200`–`5000`, xlarge) exist for page-scale layout, alongside
border-radius primitives (`--size-border-radius-radius-*`) and stroke-width
primitives (`--size-border-stroke-stroke-*`).
</div>

<div className="sb-ds-doc-section">
### Semantic tokens

Semantic tokens (`design-system/src/tokens/_spacing_semantics.scss`, matching
the **size / semantics** Figma collection) name spacing by *where it is used*,
not by its value. Each context provides padding, gap, radius, and border tokens
in `sm` / `md` / `lg` steps:

<SpacingStories.SemanticTokens />

Radius and stroke follow the same pattern: `--size-card-radius-{sm|md|full}`,
`--size-element-radius-{sm|md|lg|full}`, `--size-element-stroke-{sm|md|lg|xl}`, etc.
</div>

<div className="sb-ds-doc-section">
### Which tier belongs where

Spacing is layered from the page inward. Choose the token tier that matches the
layer you are working in:

<SpacingStories.TierGuide />

Two rules from the layout pattern guide:

- `gap: var(--size-section-gap-md)` — spacing **between major page blocks** (like two different Cards).
- `gap: var(--size-element-gap-md)` — spacing **between elements inside a Card** (like headers and text).
- **Never** use raw pixel strings like `gap: '16px'`.
</div>
